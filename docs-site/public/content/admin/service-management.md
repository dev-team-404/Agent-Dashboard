# 서비스 관리

시스템 관리자(System Admin) 관점에서 조직 내 서비스를 관리하는 방법을 안내합니다.

## 서비스란?

서비스는 API 호출의 기본 단위입니다. 각 서비스에는 고유한 **Service ID**가 부여되며, API 요청 시 `x-service-id` 헤더로 식별됩니다.

> **참고**: 서비스 생성은 누구나 할 수 있습니다. 시스템 관리자는 부서 내 서비스를 전체적으로 관리(조회, 수정, 삭제)하는 역할입니다.

## 시스템 관리자의 서비스 관리 역할

| 역할 | 설명 |
|------|------|
| 서비스 전체 조회 | dept 내 모든 서비스 목록 조회 |
| 서비스 수정/삭제 | dept 내 서비스 설정 변경 및 삭제 |
| 서비스 모니터링 | 서비스별 사용량, 멤버 현황 확인 |
| Rate Limit 관리 | 서비스/사용자별 토큰 제한 설정 |

> **참고**: 서비스 내 모델/멤버/Rate Limit 관리는 서비스 OWNER와 서비스 관리자(Service Admin)도 직접 수행할 수 있습니다.

## 서비스 생성 흐름

서비스는 일반 사용자가 직접 생성합니다. 전체 흐름은 다음과 같습니다.

```
1. 시스템 관리자가 LLM 모델 등록 및 공개 범위 설정
2. 사용자가 서비스 생성 (내 서비스 → 새 서비스 만들기)
3. 사용자가 공개된 LLM 모델을 서비스에 추가
4. 서비스 OWNER/서비스 관리자가 멤버 관리, Rate Limit 설정
5. 서비스 OWNER가 서비스 배포 (서비스 마켓 공개)
```

> 서비스 생성에 대한 자세한 내용은 [서비스 등록 가이드](/docs/user/service-registration)를 참고하세요.

## 서비스 조회

### 전체 서비스 목록

Dashboard 좌측 메뉴에서 **서비스 관리**를 클릭하면 dept 내 모든 서비스 목록을 볼 수 있습니다.

### 서비스 상세 정보

각 서비스를 클릭하면 아래 정보를 확인할 수 있습니다.

| 항목 | 설명 |
|------|------|
| 서비스 코드 / 이름 | 서비스 식별 정보 |
| 서비스 타입 | STANDARD / BACKGROUND |
| 상태 | DEVELOPMENT / DEPLOYED |
| 배포 범위 | ALL / BUSINESS_UNIT / TEAM |
| 등록자 | 서비스를 생성한 사용자 |
| 모델 목록 | 서비스에 추가된 LLM 모델 및 가중치 |
| 멤버 목록 | 서비스 멤버 및 역할 (소유자/서비스 관리자/사용자) |

## 서비스 수정

시스템 관리자는 dept 내 서비스의 정보를 수정할 수 있습니다.

1. **서비스 관리** 페이지에서 해당 서비스 클릭
2. 수정할 항목 편집
3. **저장** 클릭

> **주의**: Service ID(코드)는 등록 후 변경할 수 없습니다.

## 서비스 삭제

시스템 관리자는 dept 내 서비스를 삭제할 수 있습니다.

1. **서비스 관리** 페이지에서 해당 서비스 클릭
2. **삭제** 버튼 클릭
3. 확인 다이얼로그에서 삭제 확인

> **주의**: 서비스를 삭제하면 해당 Service ID로의 모든 API 호출이 즉시 차단되고, 관련 멤버/모델 연결/Rate Limit 설정도 함께 삭제됩니다.

## Rate Limit 관리

Rate Limit은 **시스템 관리자**, **서비스 OWNER**, **서비스 관리자** 모두 설정할 수 있습니다.

- 시스템 관리자: 사용자 관리 페이지에서 모든 서비스의 Rate Limit 설정 가능
- 서비스 OWNER/서비스 관리자: 내 서비스 페이지에서 자기 서비스의 Rate Limit 설정 가능

### 서비스 공통 Rate Limit

개별 설정이 없는 모든 사용자에게 적용되는 기본 토큰 제한입니다.

### 사용자별 Rate Limit

특정 사용자에게만 적용되는 개별 제한입니다. 사용자별 제한이 서비스 공통 제한보다 우선합니다.

### 기간 설정

| 기간 | 설명 |
|------|------|
| 5시간 (FIVE_HOURS) | 5시간 윈도우 기준 토큰 제한 |
| 24시간 (DAY) | 24시간 윈도우 기준 토큰 제한 |

## LLM 모델과 서비스의 관계

```
시스템 관리자: LLM 모델 등록 + 공개 범위 설정 (ALL / 사업부 / 팀)
         ↓
사용자: 공개된 모델을 자신의 서비스에 추가
         ↓
사용자: 모델별 가중치(1~10) 설정 → 라운드 로빈 로드 밸런싱
```

- 시스템 관리자가 LLM을 등록하고 공개 범위를 설정합니다.
- 사용자는 공개 범위에 해당하는 모델만 자신의 서비스에 추가할 수 있습니다.
- 시스템 관리자가 LLM의 공개 범위를 변경하면 해당 모델의 접근 가능 서비스도 함께 변경됩니다.
