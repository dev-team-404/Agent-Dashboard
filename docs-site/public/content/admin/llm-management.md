# LLM 관리

Dashboard에서 LLM 모델을 등록하고 관리하는 방법을 안내합니다.

## LLM 등록

### 등록 절차

1. Dashboard 좌측 메뉴에서 **LLM 모델** 클릭
2. **새 모델 추가** 버튼 클릭
3. 모델 정보 입력
4. **저장** 클릭

### 입력 항목

| 항목 | 필수 | 설명 |
|------|------|------|
| 모델명 | O | API에서 사용할 모델 이름 (예: `gpt-4o`) |
| 표시명 | O | Dashboard에서 보여지는 이름 (예: `GPT-4o`) |
| 엔드포인트 URL | O | LLM API 엔드포인트 주소 |
| 공개 범위 | O | 모델 접근 범위 설정 |
| 모델 타입 | O | CHAT / IMAGE / EMBEDDING / RERANKING |
| 설명 | - | 모델 용도 설명 |

## 공개 범위 설정

LLM 모델의 접근 범위를 설정하여 사용 권한을 제어할 수 있습니다.

| 공개 범위 | 설명 |
|-----------|------|
| **PUBLIC** | 모든 사용자/서비스가 사용 가능 |
| **BUSINESS_UNIT** | 같은 사업부 내 사용자/서비스만 사용 가능 |
| **TEAM** | 같은 팀 내 사용자/서비스만 사용 가능 |
| **ADMIN_ONLY** | 슈퍼관리자만 사용 가능 (시스템 관리자는 visibility scope 밖이면 접근 불가) |
| **SUPER_ADMIN_ONLY** | 슈퍼관리자만 사용 가능 |

### 공개 범위 선택 가이드

```
PUBLIC            — 조직 전체에서 사용하는 범용 모델
BUSINESS_UNIT     — 사업부 단위로 제한된 모델
TEAM              — 팀 내부 전용 모델
ADMIN_ONLY        — 슈퍼관리자 전용 / 테스트 중인 모델
SUPER_ADMIN_ONLY  — 시스템 내부용 모델
```

> **팁**: 새 모델을 등록할 때는 `ADMIN_ONLY`로 먼저 테스트한 후, 안정성이 확인되면 범위를 넓히는 것을 권장합니다.

## 로드밸런싱 (서비스 모델 설정)

로드밸런싱은 **서비스 모델 설정** 페이지에서 구성합니다. LLM 관리 페이지에서는 모델을 등록만 하고, 실제 라운드로빈 분배는 각 서비스 단위로 설정합니다.

### 동작 방식

```
서비스별 모델 설정 (alias 그룹)
  ├── 표시 모델명: "gpt-4o"
  │     ├── 실제 모델 A (가중치: 2x)
  │     └── 실제 모델 B (가중치: 1x)
  │     → 요청 분배: A, A, B, A, A, B ...
  └── 표시 모델명: "claude"
        └── 실제 모델 C (가중치: 1x)
        → 요청 100% 모델 C로 전달
```

- **표시 모델명(alias)**을 만들고, 그 안에 실제 LLM 모델들을 가중치와 함께 배치합니다.
- `v1/models` API는 표시 모델명만 반환합니다.
- 클라이언트가 표시 모델명으로 요청하면, 내부적으로 가중치 기반 라운드로빈으로 실제 모델에 분배됩니다.

자세한 내용은 [서비스 모델 관리](/docs/service/service-models)를 참고하세요.

## LLM 수정/삭제

### 수정

1. **LLM 모델** 페이지에서 해당 모델의 **수정** 버튼 클릭
2. 수정할 항목 편집
3. **저장** 클릭

### 삭제

1. **LLM 모델** 페이지에서 해당 모델의 **삭제** 버튼 클릭
2. 확인

> **주의**: LLM을 삭제하면 해당 모델을 사용 중인 모든 서비스에서 즉시 해당 모델을 사용할 수 없게 됩니다.

### 권한 범위

- 시스템 관리자는 **자신이 등록한 LLM**만 수정/삭제할 수 있습니다.
- 다른 시스템 관리자가 등록한 LLM은 조회만 가능합니다.
- 공개 범위가 `PUBLIC`인 모델이라도, 등록한 시스템 관리자만 설정을 변경할 수 있습니다.
- 시스템 관리자는 **자신의 visibility scope(공개 범위)에 해당하는 모델만** 조회할 수 있습니다. 예를 들어, 다른 팀 전용(`TEAM`)이나 다른 사업부 전용(`BUSINESS_UNIT`) 모델은 목록에 표시되지 않습니다.
- 슈퍼관리자는 공개 범위와 관계없이 모든 모델을 조회하고 관리할 수 있습니다.

## 엔드포인트 헬스체크

등록된 LLM 엔드포인트의 상태를 확인할 수 있습니다.

### 헬스체크 정보

| 상태 | 의미 |
|------|------|
| **정상** (초록) | 엔드포인트가 응답 가능한 상태 |
| **지연** (노랑) | 응답 시간이 느린 상태 |
| **오류** (빨강) | 엔드포인트가 응답하지 않는 상태 |

### 확인 방법

1. **LLM 모델** 페이지에서 각 모델의 **Health Check** 버튼 클릭
2. 응답 시간과 상태 확인 가능
