# 서비스 모델 관리

서비스에서 사용할 LLM 모델을 설정하고, 표시 모델명(alias)과 가중치 기반 라운드로빈을 구성하는 방법을 안내합니다.

## 서비스 모델 설정이란?

서비스 모델 설정은 **v1/models API에 노출될 가상의 모델명(표시 모델명)**을 만들고, 그 안에 실제 LLM 모델들을 배치하는 기능입니다.

```
서비스: my-chatbot
  ├── 표시 모델명: "gpt-4o"           ← v1/models에 노출
  │     ├── GPT-4o (가중치 2x)        ← 내부 라운드로빈
  │     └── GPT-4o-mini (가중치 1x)   ← 내부 라운드로빈
  └── 표시 모델명: "claude"           ← v1/models에 노출
        └── Claude Sonnet 4.6 (가중치 1x)
```

### 핵심 개념

| 용어 | 설명 |
|------|------|
| **표시 모델명 (alias)** | v1/models API 응답에 노출되는 가상 모델 이름 |
| **alias 그룹** | 하나의 표시 모델명에 연결된 실제 LLM 모델들의 묶음 |
| **가중치 (weight)** | 라운드로빈 시 해당 모델이 호출되는 비율 (1~10) |
| **라운드로빈** | alias 그룹 내 모델들이 가중치 비율로 순환 호출되는 방식 |

## 모델 설정 페이지 접근

1. **서비스 관리** 페이지에서 설정할 서비스 찾기
2. **모델 설정** 버튼 클릭
3. 서비스 모델 설정 페이지 진입

## 표시 모델명(alias 그룹) 만들기

### 새 그룹 생성

1. 하단의 **+ 새 표시 모델 추가** 버튼 클릭
2. 표시 모델명 입력 (예: `gpt-4o`, `claude`, `my-custom-model`)
3. **만들기** 클릭
4. 모델 선택 드롭다운에서 실제 LLM 모델 선택
5. **추가** 클릭

> **팁**: 표시 모델명은 자유롭게 지정할 수 있습니다. 클라이언트가 API 호출 시 이 이름으로 요청합니다.

### 그룹에 모델 추가

1. 해당 그룹의 **+ 모델 추가** 버튼 클릭
2. 타입 필터 (전체/채팅/이미지/임베딩/리랭킹) 선택
3. 모델 드롭다운에서 추가할 모델 선택
4. 가중치 설정 (기본값 1)
5. **추가** 클릭

### 그룹 이름 변경

1. 그룹 헤더의 연필 아이콘 클릭
2. 새 이름 입력
3. Enter 또는 체크 아이콘 클릭

> **주의**: 이름 변경 시 기존 모델이 삭제 후 재생성됩니다. 클라이언트가 이전 이름으로 요청하면 모델을 찾을 수 없습니다.

### 그룹 삭제

1. 그룹 헤더의 휴지통 아이콘 클릭
2. 확인 다이얼로그에서 삭제 확인

## 가중치 기반 라운드로빈

### 가중치 설정

각 모델의 `-` / `+` 버튼으로 가중치(1~10)를 조절합니다.

### 라운드로빈 동작 방식

같은 alias 그룹에 2개 이상의 활성 모델이 있으면 라운드로빈이 자동으로 적용됩니다.

```
예시: "gpt-4o" 그룹
  ├── 모델 A (가중치 2x)
  └── 모델 B (가중치 1x)

요청 순서: A → A → B → A → A → B → ...
1사이클 = 3회 (2+1)
```

- 가중치가 높을수록 더 많은 요청이 해당 모델로 분배됩니다.
- 비활성화된 모델은 라운드로빈에서 제외됩니다.
- 라운드로빈 요약은 각 그룹 하단에 자동으로 표시됩니다.

### 모델 활성화/비활성화

토글 아이콘을 클릭하면 해당 모델을 라운드로빈에서 일시적으로 제외할 수 있습니다. 비활성화된 모델은 반투명하게 표시됩니다.

## 모델 접근 권한과 접근 불가 표시

### 접근 권한 검증

모델 추가 시 본인의 부서/사업부 기준으로 접근 가능한 모델만 드롭다운에 표시됩니다.

| 공개 범위 | 접근 가능 대상 |
|-----------|---------------|
| **PUBLIC** | 모든 사용자 |
| **BUSINESS_UNIT** | 지정된 사업부에 속한 사용자 |
| **TEAM** | 지정된 팀에 속한 사용자 |
| **ADMIN_ONLY** | 슈퍼관리자(SUPER_ADMIN)만 접근 가능. 시스템 관리자는 visibility scope 밖이면 접근 불가 |
| **SUPER_ADMIN_ONLY** | 슈퍼관리자만 |

### 접근 불가 모델 (빨간색 표시)

모델이 서비스에 추가된 이후 관리자가 공개 범위를 변경하면, 해당 모델에 접근할 수 없게 될 수 있습니다. 이 경우:

- 모델이 **빨간색 배경**과 **취소선**으로 표시됩니다.
- **"접근 불가"** 뱃지가 나타납니다.
- 가중치 조절과 활성/비활성 토글이 비활성화됩니다.
- 라운드로빈에서 자동 제외됩니다.

> **대응 방법**: 관리자에게 해당 모델의 공개 범위 변경을 요청하거나, 접근 불가 모델을 제거하세요.

## v1/models API 동작

서비스에 모델이 설정되어 있으면, `v1/models` API는 **표시 모델명(alias)**만 반환합니다.

### 서비스 모델이 설정된 경우

```bash
curl http://a2g.samsungds.net:8090/v1/models \
  -H "x-service-id: my-chatbot" \
  -H "x-user-id: hong.gildong" \
  -H "x-dept-name: SW혁신팀(S.LSI)"
```

```json
{
  "object": "list",
  "data": [
    { "id": "gpt-4o", "object": "model", "created": 1709827200, "owned_by": "agent-dashboard" },
    { "id": "claude", "object": "model", "created": 1709827200, "owned_by": "agent-dashboard" }
  ]
}
```

위 예시에서 `gpt-4o`와 `claude`는 표시 모델명입니다. 실제 내부 모델이 아닌, 서비스에서 설정한 alias가 반환됩니다.

### 서비스 모델이 설정되지 않은 경우

서비스에 모델을 아직 설정하지 않았으면, 접근 가능한 전체 LLM 모델 목록이 반환됩니다 (기존 동작과 동일).

## 프록시 인증과 모델 매칭

API 호출 시 프록시 서버는 아래 순서로 모델을 매칭합니다.

### 매칭 흐름

```
1. 요청 수신
   └── body.model = "gpt-4o" (클라이언트가 보낸 모델명)

2. alias 매칭 (우선)
   └── ServiceModel에서 aliasName = "gpt-4o"인 항목 검색
   └── 매칭되면 → 해당 그룹 내 라운드로빈으로 실제 모델 선택

3. 모델명 매칭 (폴백)
   └── alias가 없으면 model.name 또는 model.displayName으로 검색

4. Rate Limit 확인

5. 프록시 전달
   └── 선택된 실제 LLM 엔드포인트로 요청 전달
```

### 인증 실패 시 응답

| 상황 | HTTP 상태 | 에러 메시지 |
|------|-----------|------------|
| 서비스 ID 없음 | 400 | Missing x-service-id header |
| 서비스 미등록 | 404 | Service not found |
| 서비스 비활성화 | 403 | Service is disabled |
| 모델 미등록 | 403 | Model not available for this service |
| Rate Limit 초과 | 429 | Rate limit exceeded |

## 모델 제거

1. 해당 모델의 휴지통 아이콘 클릭
2. 확인 다이얼로그에서 삭제 확인

> **주의**: 모델을 제거하면 해당 서비스에서 그 모델로의 API 호출이 즉시 영향을 받습니다. alias 그룹에 다른 모델이 남아있으면 라운드로빈이 재조정됩니다.

## 모델 타입

서비스에 추가할 수 있는 모델은 아래 타입을 가집니다.

| 모델 타입 | 설명 | API 엔드포인트 |
|-----------|------|---------------|
| **CHAT** | 채팅 완성 모델 | `/v1/chat/completions` |
| **IMAGE** | 이미지 생성 모델 | `/v1/images/generations` |
| **EMBEDDING** | 임베딩 모델 | `/v1/embeddings` |
| **RERANKING** | 리랭킹 모델 | `/v1/rerank` |
