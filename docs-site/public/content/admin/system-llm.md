# 레지스트리 LLM 관리

## 개요

**레지스트리 LLM 관리**에서는 Agent Registry 시스템이 AI 기능(서비스 목표 M/M 자동 추정 등)에 사용할 LLM 모델을 선택합니다.

> 💡 이 설정은 **SUPER_ADMIN**만 변경할 수 있습니다.

**사이드바**: 슈퍼 관리자 > **레지스트리 LLM 관리**

## 시스템 LLM 선택

### 요구 사항

- 등록된 LLM 모델 중 **CHAT 타입**이면서 **활성화(enabled)** 된 모델만 선택 가능
- 선택한 모델이 삭제되거나 비활성화되면 경고가 표시됩니다

### 설정 방법

1. 슈퍼 관리자 > 레지스트리 LLM 관리 페이지 접근
2. 드롭다운에서 CHAT 모델 선택
3. **저장** 클릭
4. 변경 사항은 감사 로그에 기록됩니다

## AI 추정 자동 실행 (크론)

시스템 LLM이 설정되면, **매일 자정(KST)**에 다음 프로세스가 자동 실행됩니다:

1. **배포된(DEPLOYED) 서비스** 목록 조회
2. 서비스별 **최근 5영업일 평균 DAU + API 호출 수** 산출
   - STANDARD 서비스: 실제 DAU
   - BACKGROUND 서비스: STANDARD 기준으로 추정 DAU
   - 주말/공휴일 제외
3. 서비스 설명, DAU, 호출 수를 LLM에 전달하여 **M/M 절감 효과 추정**
4. 결과를 DB에 저장 (90일 보관)

### 수동 실행

**지금 실행** 버튼으로 즉시 AI 추정을 실행할 수 있습니다. 처리 건수와 오류 건수가 표시됩니다.

## AI 추정 결과

추정 결과는 **서비스 목표 관리** 페이지의 **AI 추정** 컬럼에 표시됩니다:

- **추정 M/M**: AI가 판단한 현재 절감 효과
- **신뢰도**: HIGH(🟢) / MEDIUM(🔵) / LOW(🟡)
- **근거**: 클릭하면 AI의 추정 근거(한국어)를 확인할 수 있습니다

## 통합 대시보드 연동

통합 대시보드 상단에 두 가지 차트가 표시됩니다:

1. **부서별 Saved M/M 분포** — 도넛 차트
2. **연간 M/M 목표 달성 현황** — 올해 목표(7,000 M/M) 대비 서비스별 목표 합산, 실제 Saved, AI 추정 비교
