# 사용자/권한 관리

Dashboard에서 사용자 목록을 확인하고 권한을 관리하는 방법을 안내합니다.

## 사용자 목록 확인

### 목록 조회

1. Dashboard 좌측 메뉴에서 **사용자 관리** 클릭
2. 사용자 목록이 표시됩니다.

> **범위**: 슈퍼관리자는 **전체 사용자**를 조회할 수 있고, 시스템 관리자는 **같은 팀(dept) 소속 사용자만** 조회할 수 있습니다.

### 사용자 정보

각 사용자에 대해 아래 정보를 확인할 수 있습니다.

| 항목 | 설명 |
|------|------|
| 사용자 ID | 사용자 고유 식별자 |
| 이름 | 사용자 이름 |
| 부서(dept) | 소속 부서 |
| 역할 | 사용자, 시스템 관리자, 슈퍼관리자 |
| 최근 활동 | 마지막 API 호출 시간 |

### 검색 및 필터

- **검색**: 사용자 ID 또는 이름으로 검색
- **필터**: 역할(시스템 관리자/사용자)별 필터링 가능

## 시스템 관리자 / 슈퍼관리자 지정

일반 사용자에게 시스템 관리자(System Admin) 또는 슈퍼관리자(Super Admin) 권한을 부여할 수 있습니다. 역할 변경은 **슈퍼관리자만** 수행할 수 있습니다.

### 지정 절차

1. **사용자 관리** 페이지에서 대상 사용자 찾기
2. 해당 사용자의 **역할** 항목 클릭
3. **시스템 관리자** 또는 **슈퍼관리자**로 변경
4. **저장** 클릭

> **참고**: 슈퍼관리자(SUPER_ADMIN) 역할도 사용자 관리 페이지에서 지정할 수 있습니다. 슈퍼관리자는 요청 로그, 감사 로그, 휴일 관리 등 시스템 전체 관리 권한을 갖습니다.

### 시스템 관리자가 할 수 있는 것

시스템 관리자로 지정된 사용자는 아래 기능을 사용할 수 있습니다.

- LLM 모델 등록, 수정, 삭제
- dept 내 서비스 조회, 수정, 삭제
- 사용자 권한 관리
- 통계 조회
- Rate Limit 관리

> **참고**: 서비스 내 모델/멤버/Rate Limit 관리는 별도의 "서비스 관리자" 역할이 담당합니다. 서비스 관리자는 서비스 OWNER가 지정합니다. 자세한 내용은 [서비스 등록 가이드](/docs/user/service-registration)를 참고하세요.

## 시스템 관리자 해제

시스템 관리자 권한을 해제하여 일반 사용자로 변경할 수 있습니다.

### 해제 절차

1. **사용자 관리** 페이지에서 대상 시스템 관리자 찾기
2. 해당 사용자의 **역할** 항목 클릭
3. **사용자**로 변경
4. **저장** 클릭

> **주의**: 시스템 관리자 권한을 해제하면 해당 사용자가 등록한 LLM은 유지되지만, 더 이상 수정/삭제할 수 없게 됩니다.

## 권한 체계

Dashboard의 권한 체계는 아래와 같습니다.

### 시스템 역할 비교

| 기능 | 사용자 | 시스템 관리자 | 슈퍼관리자 |
|------|--------|-------------|-----------|
| 대시보드 접속 | O | O | O |
| 본인 사용량 확인 | O | O | O |
| 서비스 생성/관리 (본인) | O | O | O |
| LLM 등록/관리 | X | O (visibility scope 내) | O (전체) |
| 사용자 목록 조회 | X | O (팀만) | O (전체) |
| 사용자 권한 관리 | X | X | O |
| dept 내 통계 조회 | X | O | O |
| 시스템 관리자 / 슈퍼관리자 지정 | X | X | O |
| 요청 로그 조회 | X | X | O |
| 감사 로그 조회 | X | X | O |
| 휴일 관리 | X | X | O |

### 서비스 역할 비교 (서비스 내)

| 기능 | 사용자 | 서비스 관리자 | 소유자 |
|------|--------|-------------|--------|
| API 호출 | O | O | O |
| 모델 추가/제거 | X | O | O |
| 멤버 관리 | X | O | O |
| Rate Limit 설정 | X | O | O |
| 배포/배포 취소 | X | X | O |
| 서비스 삭제 | X | X | O |

### 권한 범위

- 시스템 관리자의 관리 권한은 **자신이 속한 dept 내**로 제한됩니다.
- 시스템 관리자는 **같은 팀 소속 사용자만** 조회할 수 있고, 슈퍼관리자는 **모든 사용자**를 조회할 수 있습니다.
- 시스템 관리자는 자신의 visibility scope 밖의 LLM 모델을 조회할 수 없습니다.
- 요청 로그, 감사 로그, 휴일 관리는 **슈퍼관리자 전용**이며, 시스템 관리자에게는 해당 메뉴가 표시되지 않습니다.
- 시스템 관리자가 등록한 LLM은 해당 시스템 관리자만 수정/삭제할 수 있습니다.

```
dept: SW혁신팀(S.LSI)
  ├── 시스템 관리자 A
  │     └── LLM 모델 X (시스템 관리자 A만 수정 가능)
  ├── 시스템 관리자 B
  │     └── LLM 모델 Y (시스템 관리자 B만 수정 가능)
  └── 사용자 C
        ├── 서비스 1 (OWNER로서 관리)
        └── 본인 사용량 확인
```
