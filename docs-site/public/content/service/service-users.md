# 서비스 사용자 관리

서비스에 멤버를 추가/제거하고, 역할을 관리하는 방법을 안내합니다.

## 멤버 역할

서비스에 속한 사용자는 3가지 역할 중 하나를 가집니다.

| 역할 | 설명 |
|------|------|
| **OWNER** | 서비스 생성자. 모든 관리 권한 보유 |
| **ADMIN** | 서비스 관리자. OWNER가 부여하며, 멤버/모델/설정 관리 가능 |
| **USER** | 일반 멤버. API 호출만 가능 |

### 역할별 권한 비교

| 기능 | OWNER | ADMIN | USER |
|------|-------|-------|------|
| API 호출 | O | O | O |
| 서비스 정보 수정 | O | O | X |
| 모델 추가/제거 | O | O | X |
| 멤버 추가/제거 | O | O | X |
| 멤버 역할 변경 | O | O (USER까지) | X |
| 서비스 배포/배포 취소 | O | O | X |
| Rate Limit 설정 | O | O | X |
| 서비스 삭제 | O | X | X |

### OWNER 역할

- 서비스 생성 시 자동으로 부여됨
- **서비스당 1명**만 존재
- OWNER 역할은 양도할 수 없음
- 서비스 삭제 권한은 OWNER만 보유

### ADMIN 역할

- OWNER 또는 다른 ADMIN이 부여 가능
- 서비스 삭제를 제외한 대부분의 관리 작업 가능
- 여러 명의 ADMIN 지정 가능

### USER 역할

- 서비스에 멤버로 추가되면 기본 역할
- API 호출 권한만 보유
- Dashboard에서 본인 사용량 확인 가능

## 멤버 추가하기

### 추가 절차

1. **내 서비스** 페이지에서 서비스 선택
2. 서비스 상세 화면에서 **멤버** 탭 확인
3. **멤버 추가** 버튼 클릭
4. 사용자 검색 (아래 참조)
5. 추가할 사용자 선택
6. **추가** 클릭

### 사용자 검색

멤버 추가 시 사용자를 검색하여 추가합니다.

- **검색 대상**: 시스템에 등록된 모든 사용자
- **검색 필드**: 사용자 ID(loginid) 또는 이름(username)
- **최소 입력**: 2자 이상 입력 시 검색 시작

```
검색 예시:
  "hong"     → hong.gildong, hong.sejong, ...
  "길동"     → hong.gildong (홍길동), ...
```

> **참고**: 이미 서비스 멤버인 사용자는 검색 결과에 표시되지 않습니다.

## 멤버 제거하기

1. 서비스 상세 화면에서 **멤버** 탭 확인
2. 제거할 멤버의 **삭제** 아이콘 클릭
3. 확인 다이얼로그에서 삭제 확인

### 제거 시 주의사항

- **OWNER는 제거할 수 없습니다** -- 서비스 삭제 시에만 OWNER가 함께 제거됩니다
- 멤버를 제거하면 해당 사용자의 서비스 내 Role 및 Rate Limit 설정이 함께 삭제됩니다
- 제거된 사용자는 해당 서비스의 API 호출 시 새로운 USER로 자동 등록될 수 있습니다

## 역할 변경하기

### 변경 절차

1. 서비스 상세 화면에서 **멤버** 탭 확인
2. 역할을 변경할 멤버의 **역할** 드롭다운 클릭
3. 새 역할 선택 (USER / ADMIN)
4. 자동 저장됨

### 역할 변경 규칙

| 현재 역할 | 변경 가능 역할 | 변경 가능 조건 |
|-----------|---------------|---------------|
| USER | ADMIN | OWNER 또는 ADMIN이 변경 |
| ADMIN | USER | OWNER 또는 ADMIN이 변경 |
| OWNER | - | 변경 불가 |

> **참고**: OWNER 역할은 시스템에서 자동 부여되며, 수동으로 변경하거나 양도할 수 없습니다.

## 멤버 자동 등록

서비스의 API를 호출하면 해당 사용자가 자동으로 서비스 멤버에 등록됩니다.

### 자동 등록 흐름

```
1. 사용자가 API 호출 (x-service-id: my-chatbot)
2. 서비스에 해당 사용자가 멤버로 존재하는지 확인
3. 멤버가 아닌 경우 → USER 역할로 자동 등록
4. 이미 멤버인 경우 → lastActive 시간 업데이트
```

- 자동 등록된 사용자의 기본 역할은 **USER**
- 수동으로 추가한 멤버와 동일한 방식으로 관리 가능

## 멤버 목록 확인

서비스 상세 화면의 **멤버** 탭에서 전체 멤버 목록을 확인할 수 있습니다.

### 표시 항목

| 항목 | 설명 |
|------|------|
| 사용자 ID | loginid |
| 이름 | username |
| 부서 | deptname |
| 역할 | OWNER / ADMIN / USER |
| 요청 수 | 해당 서비스에서의 총 API 호출 수 |
| 최근 활동 | 마지막 API 호출 시간 |

### 멤버 역할 아이콘

목록에서 각 역할은 아이콘으로 구분됩니다:

- **OWNER**: 왕관 아이콘
- **ADMIN**: 방패 아이콘
- **USER**: 사람 아이콘
