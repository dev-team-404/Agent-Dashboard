# 서비스 등록 가이드

누구나 Dashboard에서 서비스를 생성하고, LLM 모델을 추가하고, 배포할 수 있습니다. 이 가이드에서는 서비스 생성부터 배포까지 전체 과정을 안내합니다.

> **핵심 구조**: **시스템 관리자**(System Admin)가 LLM 모델을 등록하고 공개 범위를 설정하면, 일반 사용자는 공개된 LLM을 자신의 서비스에 추가하여 API를 호출합니다.

> **용어 안내**: 이 문서에서 "시스템 관리자"는 부서 단위의 전체 관리 권한(LLM 등록, 통계 등)을 가진 역할이고, "서비스 관리자"는 특정 서비스 내에서 모델/멤버/Rate Limit을 관리하는 역할입니다.

## 권한 구조

서비스에는 3단계 역할이 있습니다. 서비스 관리자(Service Admin)는 시스템 관리자(System Admin)와 별개의 역할입니다.

| 역할 | 한국어 | 권한 | 지정 권한 |
|------|--------|------|-----------|
| **OWNER** | 소유자 | 모델 추가/제거, 멤버 관리, Rate Limit 설정, 배포/배포 취소, 서비스 삭제 등 모든 권한 | 서비스 생성 시 자동 부여 |
| **ADMIN** | 서비스 관리자 | 모델 추가/제거, 멤버 관리, Rate Limit 설정 가능. 배포/삭제 불가 | OWNER가 지정 |
| **USER** | 사용자 | API 호출만 가능. 서비스 설정 변경 불가 | OWNER/ADMIN이 지정, 또는 API 호출 시 자동 등록 |

- 서비스를 생성한 사람이 자동으로 **OWNER**가 됩니다.
- OWNER가 다른 사용자를 **서비스 관리자(ADMIN)** 로 지정할 수 있습니다.
- API를 호출한 사용자는 자동으로 **USER**로 등록됩니다.

## 1단계: 서비스 생성

누구나 서비스를 만들 수 있습니다.

### 생성 절차

1. Dashboard 좌측 메뉴에서 **내 서비스** 클릭
2. 우측 상단 **새 서비스 만들기** 버튼 클릭
3. 서비스 정보 입력
4. **생성** 클릭

### 입력 항목

서비스 등록은 **5단계 마법사** 형태로 단계별로 진행됩니다.

| 항목 | 필수 | 단계 | 설명 |
|------|------|------|------|
| 서비스 코드 (name) | O | 1단계 | API 호출 시 `x-service-id`로 사용하는 고유 식별자 |
| 서비스 이름 (displayName) | O | 1단계 | Dashboard에서 표시되는 이름 |
| 설명 | - | 1단계 | 서비스 용도에 대한 설명 |
| 서비스 타입 | O | 2단계 | STANDARD 또는 BACKGROUND |
| 서비스 카테고리 | O | 2단계 | 7개 분류 중 택1 (예: 코드 생성 및 리뷰, 테스트 자동화 등) |
| 목표 MM | O | 3단계 | 프로젝트 목표 인력 투입량 (Men/Month) |
| 표준 M/D | △ | 3단계 | BACKGROUND 서비스의 단위 업무당 표준 공수 (Man/Day) |
| 로고 URL | - | 4단계 | 서비스 아이콘 이미지 URL |
| 서비스 URL | - | 4단계 | 서비스 바로가기 URL |
| API 문서 URL | - | 4단계 | API 연동 가이드 문서 URL |
| Jira 티켓 | - | 4단계 | 관련 Jira 티켓 URL |

> △ 표시 항목은 BACKGROUND 타입 선택 시 필수입니다.

### 서비스 코드 네이밍 규칙

서비스 코드(`name`)는 API 호출 시 `x-service-id` 헤더에 사용되는 고유 식별자입니다.

- **소문자 알파벳**, **숫자**, **하이픈(-)** 만 사용 가능
- **영문 소문자로 시작**해야 함
- 최소 3자, 최대 50자
- 공백 및 언더스코어(`_`) 사용 불가
- **중복 불가** — 이미 등록된 서비스 코드는 사용할 수 없음

```
# 올바른 예시
my-chatbot
data-pipeline-v2
code-review-bot

# 잘못된 예시
My-Chatbot       (대문자 사용)
123-service      (숫자로 시작)
my chatbot       (공백 포함)
my_chatbot       (언더스코어 사용)
ab               (3자 미만)
```

> **주의**: 서비스 코드는 생성 후 변경할 수 없습니다. 신중하게 결정하세요.

### 서비스 타입

| 타입 | 설명 | 필수 헤더 | 적합한 경우 |
|------|------|-----------|-------------|
| **STANDARD** | 사용자 식별 API 호출 | `x-service-id`, `x-user-id`, `x-dept-name` | 챗봇, 웹 앱 등 사용자 인터랙션이 있는 서비스 |
| **BACKGROUND** | 서비스 단위 API 호출 | `x-service-id`, `x-dept-name` | 배치 작업, 자동화 파이프라인, 크론 잡 |

## 2단계: 모델 추가

서비스에 LLM 모델을 추가해야 API를 호출할 수 있습니다.

### 추가 방법

1. **내 서비스**에서 해당 서비스 클릭
2. **모델** 탭 이동
3. **모델 추가** 버튼 클릭
4. 시스템 관리자가 등록하고 공개한 모델 목록에서 선택
5. 가중치(weight) 설정 후 추가

> **참고**: 모델 목록에는 시스템 관리자가 등록한 LLM 중 공개 범위(ALL / 사업부 / 팀)에 해당하는 모델만 표시됩니다. 필요한 모델이 목록에 없다면 시스템 관리자에게 LLM 공개 범위 설정을 요청하세요.

### 모델별 가중치 (Weight)

각 모델에 **1~10** 사이의 가중치를 설정할 수 있습니다. 가중치는 라운드 로빈 로드 밸런싱에 사용됩니다.

- 가중치가 높을수록 더 자주 호출됩니다.
- 예: `gpt-4o(weight=3)`, `claude-sonnet(weight=1)` → gpt-4o가 3배 더 자주 호출

### 모델 활성화/비활성화

모델을 삭제하지 않고도 **활성화/비활성화**를 전환할 수 있습니다. 비활성화된 모델은 라운드 로빈에서 제외됩니다.

---

## 3단계: 라운드 로빈 설정

서비스에 여러 모델이 추가된 경우, 라운드 로빈으로 자동 분배됩니다.

### 설정 방법

1. 서비스 상세 → **모델** 탭
2. 각 모델의 가중치(1~10) 조정
3. 드래그로 정렬 순서 변경

### 라운드 로빈 동작 원리

```
예시: 모델 3개 설정
├── gpt-4o         (weight: 3)  → 전체 요청의 50%
├── claude-sonnet  (weight: 2)  → 전체 요청의 33%
└── gemini-pro     (weight: 1)  → 전체 요청의 17%
```

- 가중치 비율에 따라 요청이 분산됩니다.
- 특정 모델을 지정하여 호출할 수도 있습니다 (`"model": "gpt-4o"`)
- 라운드 로빈은 모델명을 지정하지 않았을 때만 작동합니다

---

## 4단계: 멤버 관리

서비스에 멤버를 추가하고 역할을 부여할 수 있습니다.

### 멤버 추가

1. 서비스 상세 → **멤버** 탭
2. **멤버 추가** 버튼 클릭
3. 사용자 ID(loginid) 입력
4. 역할 선택 (ADMIN 또는 USER)
5. 추가

### 역할 변경

- OWNER/ADMIN은 다른 멤버의 역할을 변경할 수 있습니다.
- OWNER → ADMIN / ADMIN → USER 등 자유롭게 전환 가능

### 자동 등록

- API를 호출한 사용자는 **USER** 역할로 자동 등록됩니다.
- 별도 멤버 추가 없이도 API 호출이 가능합니다.

---

## 5단계: Rate Limit 설정

서비스 **OWNER** 또는 **서비스 관리자(ADMIN)** 가 서비스 전체 또는 개별 사용자에 대해 토큰 사용 제한을 설정할 수 있습니다. 시스템 관리자도 동일하게 설정 가능합니다.

### 설정 방법

1. **내 서비스** 페이지에서 서비스 선택
2. **Rate Limit** 버튼 클릭
3. 서비스 공통 제한 또는 사용자별 개별 제한 설정

### Rate Limit 종류

| 종류 | 설명 |
|------|------|
| **서비스 공통 Rate Limit** | 개별 설정이 없는 모든 사용자에게 적용되는 기본 제한 |
| **사용자별 Rate Limit** | 특정 사용자에게만 적용되는 개별 제한 |

> 사용자별 Rate Limit이 설정된 경우, 서비스 공통 제한보다 **우선 적용**됩니다.

### 기간 설정

| 기간 | 설명 |
|------|------|
| **5시간 (FIVE_HOURS)** | 5시간 윈도우 기준 토큰 제한 |
| **24시간 (DAY)** | 24시간 윈도우 기준 토큰 제한 |

## 6단계: 서비스 배포

서비스를 **서비스 마켓**에 배포하여 다른 사용자에게 공개할 수 있습니다.

### 서비스 상태

서비스는 **개발(DEVELOPMENT)** 과 **배포(DEPLOYED)** 두 가지 상태를 가지며, 자유롭게 전환할 수 있습니다.

```
서비스 생성 → [DEVELOPMENT] ⇄ [DEPLOYED]
                                  ↕ 자유 전환
```

| 상태 | 서비스 마켓 노출 | API 호출 | 설정 변경 |
|------|-----------------|----------|-----------|
| **DEVELOPMENT** | X | O (개발/테스트 가능) | O |
| **DEPLOYED** | O | O | O |

### 배포 방법

1. **내 서비스** 페이지에서 서비스 선택
2. **배포** 버튼 클릭
3. 배포 범위 선택
4. 확인

### 배포 범위 (Deploy Scope)

배포 시 서비스의 공개 범위를 지정합니다.

| 범위 | 설명 |
|------|------|
| **ALL** | 전체 조직에 공개 |
| **BUSINESS_UNIT** | 지정한 사업부에만 공개 |
| **TEAM** | 지정한 팀에만 공개 |

- BUSINESS_UNIT, TEAM 선택 시 대상 사업부/팀을 지정할 수 있습니다.
- 배포 후에도 범위를 변경할 수 있습니다.

### 배포 취소

배포된 서비스를 다시 DEVELOPMENT 상태로 되돌릴 수 있습니다.

1. 서비스 상세 화면에서 **배포 취소** 버튼 클릭
2. 서비스 마켓에서 제거됨
3. API 호출은 계속 가능
4. 언제든 다시 배포 가능

## API 호출 예시

### STANDARD 서비스

```bash
curl -X POST http://a2g.samsungds.net:8090/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "x-service-id: my-chatbot" \
  -H "x-user-id: hong.gildong" \
  -H "x-dept-name: SW혁신팀(S.LSI)" \
  -d '{"model":"gpt-4o","messages":[{"role":"user","content":"안녕하세요"}]}'
```

### BACKGROUND 서비스

```bash
curl -X POST http://a2g.samsungds.net:8090/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "x-service-id: batch-pipeline" \
  -H "x-dept-name: SW혁신팀(S.LSI)" \
  -d '{"model":"gpt-4o","messages":[{"role":"user","content":"데이터 요약해줘"}]}'
```

### 사용 가능 모델 확인

```bash
curl -X GET http://a2g.samsungds.net:8090/v1/models \
  -H "x-service-id: my-chatbot" \
  -H "x-user-id: hong.gildong" \
  -H "x-dept-name: SW혁신팀(S.LSI)"
```

## 서비스 수정 / 삭제

### 수정 가능 항목

| 항목 | 수정 가능 | 비고 |
|------|-----------|------|
| 서비스 코드 (name) | X | 생성 후 변경 불가 |
| 서비스 이름 (displayName) | O | |
| 서비스 타입 | O | STANDARD ↔ BACKGROUND 전환 가능 |
| 설명 | O | |
| 서비스 카테고리 | O | 7개 분류 중 변경 가능 |
| 목표 MM | O | |
| 표준 M/D | O | BACKGROUND 서비스에만 해당 |
| 아이콘 URL | O | 서비스 마켓에 표시 |
| 서비스 URL | O | 서비스 마켓에 표시 |
| 문서 URL | O | 서비스 마켓에 표시 |
| Jira 티켓 | O | |

### 서비스 삭제

서비스 OWNER만 삭제할 수 있습니다.

> **주의**: 서비스를 삭제하면 해당 Service ID로의 모든 API 호출이 즉시 차단되고, 관련 멤버/모델 연결/Rate Limit 설정도 함께 삭제됩니다.
