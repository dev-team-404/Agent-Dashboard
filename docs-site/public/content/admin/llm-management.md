# LLM 관리

Dashboard에서 LLM 모델을 등록하고 관리하는 방법을 안내합니다.

## LLM 등록

### 등록 절차

1. Dashboard 좌측 메뉴에서 **LLM 관리** 클릭
2. **LLM 등록** 버튼 클릭
3. 모델 정보 입력
4. **저장** 클릭

### 입력 항목

| 항목 | 필수 | 설명 |
|------|------|------|
| 모델명 | O | API에서 사용할 모델 이름 (예: `gpt-4o`) |
| 엔드포인트 URL | O | LLM API 엔드포인트 주소 |
| 공개 범위 | O | 모델 접근 범위 설정 |
| 설명 | - | 모델 용도 설명 |

## 공개 범위 설정

LLM 모델의 접근 범위를 설정하여 사용 권한을 제어할 수 있습니다.

| 공개 범위 | 설명 |
|-----------|------|
| **PUBLIC** | 모든 사용자/서비스가 사용 가능 |
| **BUSINESS_UNIT** | 같은 사업부 내 사용자/서비스만 사용 가능 |
| **TEAM** | 같은 팀 내 사용자/서비스만 사용 가능 |
| **ADMIN_ONLY** | 시스템 관리자만 사용 가능 |

### 공개 범위 선택 가이드

```
PUBLIC          — 조직 전체에서 사용하는 범용 모델
BUSINESS_UNIT   — 사업부 단위로 제한된 모델
TEAM            — 팀 내부 전용 모델
ADMIN_ONLY      — 테스트/검증 중인 모델
```

> **팁**: 새 모델을 등록할 때는 `ADMIN_ONLY`로 먼저 테스트한 후, 안정성이 확인되면 범위를 넓히는 것을 권장합니다.

## 서브모델 (로드밸런싱) 설정

하나의 모델명에 여러 엔드포인트를 등록하여 로드밸런싱을 구성할 수 있습니다.

### 서브모델이란?

동일한 모델에 대해 여러 백엔드 서버(엔드포인트)를 설정하면, 요청이 자동으로 분산 처리됩니다.

```
모델: gpt-4o
  ├── 서브모델 1: https://endpoint-a/v1 (가중치: 50%)
  ├── 서브모델 2: https://endpoint-b/v1 (가중치: 30%)
  └── 서브모델 3: https://endpoint-c/v1 (가중치: 20%)
```

### 서브모델 등록 방법

1. LLM 상세 페이지에서 **서브모델 추가** 클릭
2. 서브모델 엔드포인트 URL 입력
3. 가중치(선택) 설정
4. **저장** 클릭

### 로드밸런싱 동작 방식

- 가중치를 설정하면 해당 비율로 요청이 분배됩니다.
- 특정 서브모델이 응답 불가 상태이면 자동으로 다른 서브모델로 폴백됩니다.
- 헬스체크를 통해 각 서브모델의 상태를 모니터링할 수 있습니다.

## LLM 수정/삭제

### 수정

1. **LLM 관리** 페이지에서 해당 모델 클릭
2. 수정할 항목 편집
3. **저장** 클릭

### 삭제

1. **LLM 관리** 페이지에서 해당 모델 클릭
2. **삭제** 버튼 클릭
3. 확인

> **주의**: LLM을 삭제하면 해당 모델을 사용 중인 모든 서비스에서 즉시 해당 모델을 사용할 수 없게 됩니다.

### 권한 범위

- 시스템 관리자는 **자신이 등록한 LLM**만 수정/삭제할 수 있습니다.
- 다른 시스템 관리자가 등록한 LLM은 조회만 가능합니다.
- 공개 범위가 `PUBLIC`인 모델이라도, 등록한 시스템 관리자만 설정을 변경할 수 있습니다.

## 엔드포인트 헬스체크

등록된 LLM 엔드포인트의 상태를 확인할 수 있습니다.

### 헬스체크 정보

| 상태 | 의미 |
|------|------|
| **정상** (초록) | 엔드포인트가 응답 가능한 상태 |
| **지연** (노랑) | 응답 시간이 느린 상태 |
| **오류** (빨강) | 엔드포인트가 응답하지 않는 상태 |

### 확인 방법

1. **LLM 관리** 페이지에서 각 모델 옆 상태 표시 확인
2. 상세 페이지에서 서브모델별 헬스 상태 확인 가능

> **팁**: 헬스체크에서 오류가 지속되면 엔드포인트 서버 상태를 확인하거나, 서브모델을 통한 폴백 구성을 고려하세요.
