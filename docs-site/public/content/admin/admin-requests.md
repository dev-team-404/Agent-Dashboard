# 관리자 권한 신청/승인

## 개요

일반 사용자가 **시스템 관리자(ADMIN) 권한**을 신청하고, 기존 관리자가 이를 **승인 또는 거부**할 수 있는 기능입니다.

## 대상

각 팀 **AA(AI Agent)** 담당자들에게 다음 권한을 위한 관리자 역할을 부여합니다:

- M/M 및 Saved M/M 서비스 목표 관리
- 서비스 및 LLM 모델 관리
- 사용자 관리 및 통계 조회

## 사용자: 권한 신청

**사이드바**: 개인 > **관리자 권한 신청** (관리자가 아닌 사용자에게만 표시)

### 신청 방법

1. 관리자 권한 신청 페이지 접근
2. Super Admin 연락처 확인 (문의 필요 시)
3. **신청 사유** 작성 (5자 이상)
4. **신청하기** 버튼 클릭

### 제한 사항

- 이미 관리자인 경우 신청 불가
- 대기 중인 신청이 있으면 중복 신청 불가
- 거부된 후에는 재신청 가능

### 내 신청 내역

신청 페이지 하단에서 이전 신청 내역을 확인할 수 있습니다:

| 상태 | 설명 |
|------|------|
| 🟡 대기중 | 관리자 승인 대기 |
| 🟢 승인 | 관리자 권한이 부여됨 |
| 🔴 거부 | 거부됨 (거부 사유 확인 가능) |

## 관리자: 신청 처리

**사이드바**: 시스템 관리 > **권한 신청 관리**

### 신청 목록

- **대기중 / 전체 / 승인 / 거부** 탭으로 필터링
- 각 신청 카드에 표시되는 정보:
  - 신청자 이름, 사번(loginid), 직급
  - 소속 부서, 사업부
  - 신청 사유
  - 신청 일시

### 승인

**승인** 버튼 클릭 시:
- 해당 사용자가 **ADMIN** 역할로 즉시 등록됩니다
- 부서 정보와 사업부가 자동으로 설정됩니다
- 감사 로그에 `APPROVE_ADMIN_REQUEST`로 기록됩니다

### 거부

**거부** 버튼 클릭 시:
- 거부 사유를 입력할 수 있습니다 (선택)
- 신청자의 내역에 거부 사유가 표시됩니다
- 감사 로그에 `REJECT_ADMIN_REQUEST`로 기록됩니다

## Knox 연동

신청 시 Knox 임직원 API로 다음 정보를 자동 조회합니다:

- 한글 이름 (fullName)
- 부서명 (departmentName)
- 직급 (titleName)
- 사업부 (businessUnit)
