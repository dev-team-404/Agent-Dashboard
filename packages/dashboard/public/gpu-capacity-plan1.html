EXECUTIVE SUMMARY

GPU 용량 측정 방안

1 / 5

GPU 내부 메트릭(nvidia-smi) 없이, 기존 요청 로그만으로 GPU 사용률 / 부족분 / 미래 수요를 산출하는 방안입니다.

배경

🎯 문제

현재 GPU 인프라의 실제 사용률을 정량적으로 파악할 수 없어,
"GPU가 부족한가? 얼마나 부족한가? 언제 증설해야 하는가?"에
데이터 기반 답변이 불가능한 상황입니다.

💡 제안

이미 수집 중인 요청 로그(request_logs)의 응답 시간 변화를 분석하여
GPU 사용률을 0~100%로 실시간 측정하고,
포화 전 조기 경보 + 미래 예측까지 제공합니다.

기대 효과

📈 실시간 사용률 대시보드

"현재 GPU 사용률 65%" — 한눈에 파악

⚠️ 조기 경보 시스템

80% 초과 시 자동 알림 — 장애 발생 전 선제 대응

🔮 미래 수요 예측

"사용자 2배 증가 시 GPU 2대 추가 필요" — 예산 수립 근거

💰 추가 비용 없음

기존 로그 데이터만 활용 — 별도 모니터링 도구 불필요

🚗
핵심 아이디어: 새벽에 20분 걸리던 출근길이 40분 걸리면 도로 50% 혼잡.
GPU도 "한산할 때 대비 얼마나 느려졌는가"로 포화도를 측정합니다.

Baseline이란?
사내 서비스 특성상 새벽 트래픽이 없으므로, "최근 30일 중 RPS 하위 5% 시간대의 P95 레이턴시"를 baseline으로 사용합니다. 이는 GPU가 가장 여유로운 상태의 응답 시간으로, 순수 연산 성능을 대표합니다.
대시보드 표시 예시
오전 10시 (업무 시작)
35%
오후 2시 (평균)
65%
오후 4시 (피크)
91%
⚠ 80% 초과 시 "GPU 증설 필요" 자동 알림

산출 공식

사용률 // baseline = RPS 하위 5% 시간대 P95
// saturation = timeout 직전 P95

rate = (current_p95 - baseline)
     ÷ (saturation - baseline)

GPU 사용률(%) = rate × 100
부족분 // 레이턴시 비율로 GPU 부족분 역산

gap_ratio = 1 - baseline ÷ current_p95

부족 GPU = 보유 GPU × gap_ratio

// 예: GPU 8장, P95가 baseline의 1.6배
// gap = 1 - 1/1.6 = 0.375
// 8 × 0.375 = 3장 부족
미래 예측 // RPS 성장 추세로 미래 사용률 예측

future_RPS = current × (미래 사용자 ÷ 현재)

→ 해당 RPS에서의 P95 예측
→ 사용률 & 부족 GPU 산출

"피크 시 91%로 포화 임계점(80%)을 초과합니다.
GPU 3장 부족으로 추정되며 증설이 필요합니다."

→ 구체적 수량 근거로 증설 의사결정 지원

🔴 시나리오 3 — 미래 계획

"직원이 500명 추가되면 상시 포화 상태가 되어
GPU 서버 2대 추가가 필요합니다."

→ 사용자 증가에 따른 사전 예산 수립 근거

선행 수정 사항

🔧 로그 데이터 정비 (선행 필수)
1. Baseline 산출 방식 변경
  • 현재: 사내 서비스라 새벽 트래픽 = 0 → baseline 산출 불가
  • 해결: "최근 30일 중 RPS 하위 5% 시간대의 P95"를 baseline으로 사용

2. 스트리밍 요청 분리
  • 현재: stream=true 요청은 latencyMs에 전체 스트림 시간 포함 (10배+ 부풀림)
  • 해결: 비스트리밍 요청만 집계 (WHERE stream = false)

구현 단계

1

Baseline 자동 산출 로직

매일 새벽 크론잡으로 최근 30일 RPS 하위 5% 시간대의 P95 레이턴시를 baseline으로 저장

2

실시간 사용률 산출 API

현재 P95 대비 baseline 비율을 계산하여 0~100% 사용률 반환하는 API 엔드포인트

3

부족분 역산 로직

레이턴시 비율 기반 GPU 부족분 산출 (gap_ratio = 1 - baseline/current)

4

대시보드 시각화

게이지 차트 + 시계열 그래프 + 경보 이력을 관리 대시보드에 추가

5

경보 시스템

사용률 80% 초과 시 자동 Slack/이메일 알림 + 월간 보고서 자동 생성
